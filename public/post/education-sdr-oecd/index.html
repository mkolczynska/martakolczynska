Educational attainment: surveys vs. administrative records

10 Aug 2018, 05:13
surveys / SDR / OECD / education / R / ggplot / plotly
  • Problem
  • Data
  • Results: Scatter plots
  • Closer look at ISSP

with Przemek Powałko

tl;dr: General population surveys with representative samples should have a similar education structure as shown by data from administrative sources, especially if survey weights are used. In this post we compare cross-national survey data from the SUrvey Data Recycling project with educational attainment statistics from the OECD. We find a strong positive correlation between both datasets, with some interesting deviations that we start to investigate.

Problem

Social surveys commonly ask about respondents’ educational attainment, either about the highest completed level of education, or about the number of completed years of schooling, and sometimes both. Survey measurements of individual education can be aggregated to indicate the overall level of education in the society.
Information about the country’s level of education can also be obtained from administrative data, such as the Organisation for Economic Co-operation and Development (OECD).
For general population surveys based on representative samples one could expect survey aggregates of educational attainment to closely track data from OECD. In this note I try to find out if this is really the case.

Data

Survey aggregates come from the Survey Data Recycling database (SDR), which combines selection harmonized variables from 22 international survey projects. The data and documentation are available from Dataverse. For more information about SDR see the project website or this post.

OECD provides data on the shares of populations by educational attainment in the Educational attainment and labour-force status database.

library(OECD) # getting OECD data
library(dataverse) # getting data from Dataverse
library(tidyverse) # cleaning and reshaping data
library(countrycode) # switching between country codes

library(ggplot2) # plots
library(plotly) # interactive plots

OECD

Let’s start with the OECD data. The function get_datasets from the oecd gets the list of available datasets. search_dataset searches for keywords among the available datasets. The Educational attainment and labour-force status database has the code NEAC_SHARE_EA. With this code, the database can be downloaded with the get_dataset function.

Note: The entire database is huge and takes several minutes to download. According to the oecd package instructions, it is possible to specify a filter in get_dataset. This would be great, since I only need a small subset of the data, but unfortunately I haven’t been able to get the filetr to work.

After downloading the database I select the subset I’m interested in:
1. the indicator “Share of population by educational attainment” (NEAC_SHARE_EA), 2.the education level measured “At least upper secondary education” (L3T8), 3. year range excluding “Latest available year” (code 9999), 4. the type of the measureVALUE(as opposed to standard error,SE), 5. both sexes, 6. age range of 25-64 (Y25T64`), 7. only individual countries, i.e. excluding country groups such as the G20 or the OECD average.

dataset_list <- get_datasets()
search_dataset("education", data = dataset_list)

dataset <- "EAG_NEAC"

oecd <- get_dataset(dataset = dataset) %>% 
  filter(INDICATOR == "NEAC_SHARE_EA", 
                     ISC11A == "L3T8", 
                     obsTime != "9999",
                     MEASURE == "VALUE",
                     SEX == "T",
                     AGE == "Y25T64",
                     !COUNTRY %in% c("E22", "OAVG", "G20", "NMEC")) %>%
  mutate(iso3c = COUNTRY, year = as.numeric(obsTime)) %>%
  select(iso3c, year, obsValue)

SDR

Knowing the DOI, it is easy to get a list of all files in a given repository, including data files and documentation, with the get_dataset function. The SDR Master File contains harmonized survey variables. The stata version has the id = 3006244, which is a parameter in the get_file function to download the data file to R (see documentation for details). The data file is pretty large, so it takes a while to download.

master.raw <- get_file(3006244)
tmp <- tempfile(fileext = ".dta")
writeBin(as.vector(master.raw), tmp)
master <- haven::read_dta(tmp)

Cleaning and merging SDR and OECD data

  1. Filter the SDR Master File to include only respondents aged between 25 and 64,
  2. using the “Education level” variable (t_edu) create a boolean variable (hs) indicating whether the respondent completed upper secondary education,
  3. group by national survey (t_survey_name, t_survey_edition, t_country_l1u, t_country_set),
  4. calculate weighted proportions of the hs variable by national sample - mean_hs,
  5. calculate the proportion of missing values in the t_edu variable,
  6. filter surveys for which mean_hs is available and which cover whole country samples (as opposed to split-country samples, such as West and East Germany or Wallonia and Flanders),
  7. covert SDR country codes from ISO2 to ISO3,
  8. merge the SDR data with the OCD data on the country code and year,
  9. filter out cases with missing values.
merge <- master %>% filter(t_age >= 25 & t_age <= 64) %>%
  mutate(hs = ((t_edu == 30 & c_edu_incomplete == 0) | 
                 (t_edu > 30 & t_edu < 90)) ) %>%
  group_by(t_survey_name, t_survey_edition, t_country_l1u, t_country_set) %>%
  summarise(mean_hs = 100 * weighted.mean(hs, t_weight_l1u_2, na.rm = TRUE),
            year = mean(t_country_year),
            prop_edu_mis = mean(is.na(t_edu) | t_edu == 90)) %>%
  filter(!is.na(mean_hs), nchar(t_country_l1u) == 2) %>%
  mutate(iso2c = substr(t_country_l1u, 1, 2),
         iso3c = countrycode(iso2c, 'iso2c', 'iso3c'),
         continent = countrycode(iso2c, 'iso2c', 'continent')) %>%
  inner_join(oecd, by = c("iso3c", "year")) %>%
  filter(!is.na(obsValue))

Results: Scatter plots

A simple scatter plot will show the extent of agreement or disagreement between the survey proportion of individuals with at least upper secondary education and the data from OECD.

ggplot(merge, aes(x=obsValue, y=mean_hs)) + geom_point() + 
  ggtitle("Educational attainment from SDR and OECD") + 
  xlab("OECD: % of population with at least upper secondary education") +
  ylab("SDR: % of population with at least upper secondary education") +
  xlim(0,100) + ylim(0,100) +
  geom_abline(intercept = 0, slope = 1)

Interactive graphs make it easier to identify cases (by hovering the mouce over the data point).

plot_ly(merge , x = ~obsValue, y = ~mean_hs, color = ~continent,
        colors = "Set1", marker = list(size = 5),
        hoverinfo = 'text', 
        text = ~paste('country:', iso3c,
                      '<br>year:', year,
                      '<br>project wave:', t_survey_name, t_survey_edition)) %>%
  layout(title = "Educational attainment from SDR and OECD",
         xaxis = list(title = "OECD: Share of population by educational attainment", 
                      range = c(0,100)),
         yaxis = list(title = "SDR: Survey % of completed upper secondary education", 
                      range = c(0,100)),
         margin = list(t = 100, b = 100))

Closer look at ISSP

It seems that some of the most deviant cases far below the 45-degree line come from the International Social Survey Programme (ISSP).

merge %>% filter(t_survey_name == "ISSP" ) %>%
  ggplot(., aes(x=obsValue, y=mean_hs, col = t_survey_edition)) + 
  geom_point() + 
  ggtitle("Educational attainment from SDR and OECD: ISSP") + 
  xlab("OECD: % of population with at least upper secondary education") +
  ylab("SDR: % of population with at least upper secondary education") +
  xlim(0,100) + ylim(0,100) +
  geom_abline(intercept = 0, slope = 1)

Further investigation required digging into the original data and documentation. We decided to check ISSP surveys from Switzerland, which is an interesting case, because the survey proportions cluster around 85-89% for some ISSP waves and around 40% for others. Such large changes do not normally appear in nature.

It turns out that some ISSP waves have two variables capturing respondent’s education level: one is country-specific and the other is recoded to a common scheme across countries by ISSP. For example, in ISSP/2007, the country-specific variable for Switzerland (ch_degree) has 14 categories (including one for missing values). The degree variable is the recoded one and has 7 categories (also including one for missing values). The frequencies in both variables are nice enough to make a relatively safe bet as to how ch_degree was mapped onto degree.

We checked the SDR documentation and the code used to harmonize education variables from ISSP. It turns out that the source variable used for harmonization if ISSP/2007/CH was the recoded degree variable. Below is a chart showing how the three variables are related.

It seems that the problem lies in the recoding from ch_degree to degree, especially in the value label Lowest formal qualification, which suggests primary school, as it was coded in SDR. Meanwhile Lowest formal qualification includes both Primary school and Secondary school, which would be assigned different ISCED codes.
